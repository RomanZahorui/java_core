Если массив содержит N элементов, то среднее количество шагов, необходимых для нахождения элемента, равно N/2
В худшем случае заданный элемент находится в последней ячейке и для его обнаружения потребуется N шагов.
Таким образом, время выполнения алгоритма пропорционально количеству шагов, а поиск в среднем выполняется 
значительно медленнее (N/2 шагов) вставки (один шаг, т.к. вставка осуществляется в конец массива, в первую 
свободную ячейку). 

Удаление (при условии запрета дубликатов) требует поиска в среднем по N/2 элементам и последующего перемещения 
остальных элементов (в среднем N/2) для заполнения пустой ячейки. В сумме получается N шагов.
